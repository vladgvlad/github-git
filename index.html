Basic Git Commands

A quick guide to the most common Git commands for managing repositories.

Git Commands

1. git init

Description: Initializes a new Git repository in the current directory. It creates a hidden folder called `.git` where all Git configuration and tracking files are stored.

git init

2. git clone

Description: Clones a remote repository to your local machine. Use the repository URL to create a local copy.

git clone repository-url

3. git status

Description: Shows the status of files in the repository, indicating which ones are modified, which are ready to be committed, etc.

git status

4. git add

Description: Adds files to the staging area to be committed. You can add a specific file or all modified files.

git add file
git add .

5. git commit

Description: Records changes in the local repository. This command should be used after adding files with `git add`.

git commit -m "commit-message"

6. git push

Description: Pushes committed changes to the remote repository. A remote repository must be configured first.

git push origin branch-name

7. git checkout

Description: Switches to a different version or branch to view or work on a specific version.

git checkout branch-name

8. git branch

Description: Shows the branches in your repository. You can also use this command to create a new branch or delete an existing one.

git branch
git branch branch-name

9. git merge

Description: Merges one branch into the current branch. You need to be on the branch you want to merge into.

git merge branch-name

10. git log

Description: Displays the commit history of the repository, with detailed information about each commit.

git log

11. git remote

Description: Manages the connection with other repositories.

git remoteremote-URL
